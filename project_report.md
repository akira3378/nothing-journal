# Nothing Community Platform - 产品功能手册与技术报告

## 第一部分：用户功能指南

欢迎来到 **Nothing** —— 专属社区平台。

### 1. 快速上手

#### 注册与入驻 (Application)
Nothing 采用"邀请/申请制"的会员体系，确保社区的高质量与纯粹性。
*   **提交申请**: 在首页点击 "申请入驻" (Apply for Membership)。
*   **身份验证**: 输入您的电子邮箱，系统将发送一个一次性验证码 (OTP) 确认您的邮箱所有权。
*   **建立数字身份**:
    *   **昵称**: 设置您在社区行走的代号。
    *   **职业/身份标签**: 添加最多3个标签（如：设计师、开发者、赛博游牧民），定义您的多重身份。
    *   **头像上传**: 上传您的个性化头像。
    *   **凭证上传**: 上传您的会员资格凭证（如邀请函、作品集截图等），供管理员审核。
*   **审核流程**: 提交后，您的账户将处于 "审核中" (Pending) 状态。管理员批准后，您将获得完整访问权限。

#### 登录 (Access)
*   **无密码体验**: 我们采用 **Magic Link (魔术链接)** 技术。输入邮箱，点击邮件中的链接（或输入验证码）即可直接进入，无需记忆密码，安全且便捷。

### 2. 社区功能 (The Feed)

#### 信息流 (Information Stream)

*   **浏览动态**: 以时间流形式查看所有成员的广播。
*   **发布内容**:
    *   支持纯文本表达。
    *   支持上传多张图片（自动压缩优化）。
    *   支持标记地理位置。
*   **互动交流**:
    *   **点赞**: 对产生共鸣的内容表达喜爱。
    *   **评论与回复**: 支持多层级嵌套评论。您可以直接回复某条评论，展开深度的盖楼讨论。
    *   **删除**: 您可以随时撤回（删除）自己发布的内容或评论。

### 4. 个人中心 (Profile)

#### 数字档案
您的个人主页是您在 Nothing 世界的投射。
*   **身份展示**: 显示您的昵称、头像、加入时间、身份标签以及会员状态。
*   **我的传输**: 集中展示您发布过的所有历史动态。
*   **凭证档案**: 查看您入驻时提交的凭证记录。

#### 资料管理
*   **编辑资料**: 您可以随时更新昵称、调整身份标签，或更换新的头像。
*   **会员状态**:
    *   **Active (活跃)**: 正常使用所有功能。
    *   **Expired (过期)**: 会员资格按年度管理（统一至次年5月1日），过期后需上传新凭证进行续期。

### 5. 通知与公告
*   **实时通知**: 当有人评论您的帖子或回复您的评论时，您会收到通知。
*   **社区公告**: 首页置顶显示管理员发布的最新动态和重要通知。

## 第二部分：管理员指南 (Administration)

拥有管理员权限的用户将拥有额外的 "控制中心" (Control Center) 入口。

### 1. 成员管理 (Member Database)
*   **审核申请**: 查看新注册用户的凭证，执行 "批准" (Approve) 或 "拒绝" (Reject) 操作。
*   **用户治理**:
    *   **挂起 (Suspend)**: 暂时封禁违规用户。
    *   **恢复 (Restore)**: 解封用户。
    *   **角色变更**: 提拔普通用户为管理员。
    *   **有效期管理**: 手动调整用户的会员过期时间。

### 2. 内容与配置
*   **公告管理**: 发布、编辑或删除全站公告。
*   **站点配置**:
    *   **Logo 设置**: 自定义网站 Logo。
    *   **背景视频**: 设置首页的背景视频 (YouTube 链接)，营造独特的社区氛围。

## 第三部分：技术栈与架构细节

本平台采用现代化的全栈架构，注重性能、安全性和开发效率。

### 1. 技术栈 (Tech Stack)

#### 前端 (Frontend)
*   **核心框架**: **React 18** (TypeScript) - 保证了类型安全和组件化开发的高效性。
*   **构建工具**: **Vite** - 提供极速的开发服务器启动和生产环境构建。
*   **样式方案**: **Tailwind CSS** - 原子化 CSS 框架，实现了高度定制化且响应式的设计系统 (Dark Mode 支持)。
*   **路由管理**: **React Router v6** - 处理单页应用 (SPA) 的页面跳转。
*   **UI 组件**: 自研极简组件库 + **Lucide React** (图标)。

#### 后端与基础设施 (Backend & Infra)
*   **BaaS (Backend-as-a-Service)**: **Supabase**
    *   **Database**: PostgreSQL - 强大的关系型数据库。
    *   **Auth**: Supabase Auth - 处理 Magic Link、JWT Token 管理和用户会话。
    *   **Storage**: Supabase Storage - 对象存储，用于托管用户头像、帖子图片和凭证。
    *   **Realtime**: 基于 PostgreSQL 的实时订阅，用于通知推送和管理后台数据同步。
*   **托管 (Hosting)**: **Vercel** (推荐) - 针对前端优化的边缘网络托管。

### 2. 关键实现细节
*   **安全性**:
    *   **RLS (Row Level Security)**: 数据库层面的安全策略。例如，只有作者本人或管理员可以删除帖子；只有管理员可以查看所有用户的敏感凭证。
    *   **无密码认证**: 彻底杜绝了弱密码和撞库风险。
*   **性能优化**:
    *   **图片压缩**: 前端在上传前自动对图片进行 Canvas 压缩 (JPEG 0.7)，显著减少带宽消耗和加载时间。
    *   **虚拟列表/分页**: Feed 流采用分页加载，保证长列表浏览流畅。

## 第四部分：免费版承载能力评估报告

基于 **Supabase Free Tier** 和 **Vercel Hobby Plan** 的资源限制，以下是本系统的预估承载能力。

### 1. 资源配额限制 (Free Tier)

| 资源项 | Supabase Free | Vercel Hobby | 瓶颈分析 |
| :--- | :--- | :--- | :--- |
| **数据库容量** | 500 MB | N/A | 纯文本数据可存数百万条，非常充足。 |
| **文件存储** | 1 GB | N/A | **主要瓶颈**。若每张图压缩后 200KB，约可存 5000 张图片。 |
| **月带宽** | 5 GB | 100 GB | Supabase 带宽较紧缺，主要消耗在图片加载。 |
| **月活跃用户 (MAU)** | 50,000 | N/A | 远超一般私密社区需求。 |
| **并发连接数** | 500 (Direct) | N/A | 足够支持中型社区。 |
| **实时连接 (Realtime)** | 200 Concurrent | N/A | **硬性限制**。同时在线并接收通知的人数不能超过 200。 |

### 2. 承载量估算

#### 用户量级
*   **注册用户数**: 支持 **10,000+** 注册用户毫无压力（主要受限于数据库行数限制，但 500MB 足够存很多用户资料）。
*   **日活用户 (DAU)**: 建议控制在 **1,000 - 2,000** 人以内。

#### 并发能力
*   **同时在线人数**: 建议控制在 **200人** 以内。
    *   这是受限于 Supabase Realtime 的免费配额（200个并发客户端）。超过此限制，实时通知功能可能会失效，但普通的浏览和发帖（REST API）仍然可用。

#### 存储与带宽风险
*   **图片存储**: 1GB 空间较小。如果社区活跃，每天产生 50 张新图，大约 3-4 个月就会填满。
    *   *建议方案*: 定期清理旧数据，或升级到 Pro Plan ($25/月) 获取 100GB 存储。
*   **带宽**: 5GB 带宽如果用于高频图片浏览比较紧张。Vercel 的 100GB 带宽主要用于加载网页代码本身，图片流量主要走 Supabase。

### 3. 总结建议
此架构非常适合 **中小型私密社区、初创团队内部工具或兴趣小组（< 5000人）**。
*   **初期**: 免费版完全够用，性能体验极佳。
*   **中期**: 当存储空间接近 1GB 或同时在线人数常态化超过 200 人时，只需将 Supabase 升级至 Pro Plan 即可无缝扩容，无需重构代码。
